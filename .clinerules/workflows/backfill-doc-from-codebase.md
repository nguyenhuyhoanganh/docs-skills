# Backfill Doc From Codebase Workflow

Use skills `doc-context-discovery`, `doc-diagrams-as-code`, and phase authoring skills as needed.

Then:

1. Discover system behavior from existing code and operational artifacts.
2. Generate baseline core-4 docs (feature brief/requirements/HLD/DLD) from evidence.
3. Add optional artifacts (API/Ops/Security/Traceability) only when requested or evidence shows they are needed.
4. Verify generated docs against code evidence and revise where discrepancies are found.
5. Mark assumptions explicitly where evidence is incomplete.
6. Publish as `draft` with review tasks for owning teams.
