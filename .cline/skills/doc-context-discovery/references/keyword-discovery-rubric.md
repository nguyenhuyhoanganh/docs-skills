# Keyword Discovery Rubric

## Keyword Types

- Business terms (feature, product term, user-facing concept)
- Technical terms (service names, module names, entity names)
- API terms (endpoint nouns, event names)
- Operational terms (SLO, runbook, incident, alert)

## Minimum Keyword Set

- 8 keywords minimum
- 15 keywords maximum
- Include plural and common alias variants if relevant

## Bad Keyword Set

- Too generic: `system`, `feature`, `service`
- No domain nouns
- No code symbol names

## Good Keyword Set

- Concrete domain nouns
- Service/module identifiers
- API/event names
- Incident/operational terms where relevant
